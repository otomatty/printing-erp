# ファイル検索術

モノレポ構造のプロジェクトでは、ファイル数が多いため、目的のコンポーネントファイルを探すのに時間がかかることがあります。VSCodeで共通コンポーネントなどの実装を効率的に検索・確認するための方法をまとめました。

## ショートカットキーによるファイル検索

### 基本的なファイル検索
- `Cmd+P`（macOS）または `Ctrl+P`（Windows/Linux）
  - ファイル名の一部を入力するだけで、あいまい検索ができます
  - 例: `page-header` や `pageheader` でも検索可能

### シンボル検索（コンポーネント名から検索）
- `Cmd+T`（macOS）または `Ctrl+T`（Windows/Linux）
  - コンポーネント名（関数名やクラス名）から直接検索できます
  - 例: `PageHeader` コンポーネントを直接検索

### 定義元へのジャンプ（最も効率的な方法）
- インポート文から直接ジャンプ:
  - インポート文の `PageHeader` 部分にカーソルを合わせて
  - `Cmd+クリック`（macOS）または `Ctrl+クリック`（Windows/Linux）
  - または `F12` キーを押す
  - 例: `import { PageHeader } from '@kit/ui/page-header';` の `PageHeader` 部分にカーソルを合わせて実行

## 検索のヒント

### 検索を絞り込む
- パスの一部も含めることで検索結果を絞り込み可能
  - 例: `ui page-header` や `kit ui page` など

### エイリアスパスの理解
- `@kit/ui` のようなエイリアスが実際にどのディレクトリを指しているか把握しておく
  - この例では `packages/ui/src` を指している

## 実際の例

`@kit/ui/page-header` から PageHeader コンポーネントを探す場合：

1. インポート文から直接ジャンプする
```tsx
import { PageHeader } from '@kit/ui/page-header';
```
↑ この `PageHeader` にカーソルを合わせ、`Cmd/Ctrl+クリック` または `F12`

2. ファイル検索を使う
   - `Cmd/Ctrl+P` → `page-header` と入力

3. シンボル検索を使う
   - `Cmd/Ctrl+T` → `PageHeader` と入力

これらの方法を使い分けることで、大規模なモノレポでも目的のコンポーネントに素早くアクセスできます。
